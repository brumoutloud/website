10. Interaction & Animation Principles

Our animation philosophy is **Purposeful, Responsive, and Performant**. Animations should enhance the user experience by providing meaningful feedback, not distract from it. Motion should be swift and fluid.

STANDARD HOVER STATE (CARDS)

Image

Hover Me

Card lifts and title changes colour.

Interactive cards should lift slightly on hover to indicate they are clickable. We use a subtle `transform: translateY(-5px)`. The transition should be quick (0.3s).

FOCUS STATE (FORMS)

Essential for accessibility. All interactive elements (buttons, links, inputs) must have a clear focus state. We use a purple ring outline.

ELEMENT ENTRANCE ANIMATION

This card fades and slides in smoothly when the page loads.

When loading content onto a page (like the event grid), elements should appear with a subtle fade-and-slide-up animation to prevent jarring layout shifts.

.animate-fade-in {
  animation: fadeInUp 0.5s ease-out forwards;
}

MICRO-INTERACTION (BUTTONS)

Small, delightful animations provide satisfying feedback. On this button, the arrow icon moves slightly to the right on hover, reinforcing the "forward" action.
